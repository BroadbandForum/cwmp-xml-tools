About TRminator

Purpose:

TRminator was created for the Broadband Forum and all users of their Data Models, specifically the XML documents used for CWMP/TR-069.
TRminator enables the user to create human-readable representations of the information in the XML Data Model files.

TRminator was created using Java 6, and requires the Java Runtime Environment to run.

Quick-Start Guide:

Introduction:
TRminator is designed to parse information in a BBF XML document into an HTML table, and make a file of it.

Please Note:
in order for TRminator to function correctly, all documents referred to in the XML document, including those by recursion (document referenced in a referenced document) MUST be available in the same directory as the input file.

TRminator is run via the executable Java Jar named "TRminator.jar."
Java 6 is required to run it.
In most environments, double-clicking the jar will run it without any arguments (doing so will invoke the graphical interface of TRminator).

Running the jar on the command line allows for arguments: 
user@host> java -jar TRminator.jar [args]
If no arguments are used, the graphical interface will appear.

If arguments are to be used, the first argument is a mandatory argument setting the mode:
     "-cli" will run TRminator's command-line interface, in the present console.
     "-gui" will run TRminator's graphical interface.

     - click for a quick guide to the UI (default mode)

Following the mode can be any number of arguments, in no particular order. The graphical interface does not currently support any arguments.
The command-line interface has two mandatory arguments:
     "-i [path]" is the input option. It is to be followed by a string representing the path to the input file.
     "-o [path]" is the output option. It is to be followed by a string representing the path to the output file.

For more information and arguments for the command-line interface, see its documentation

Known Issues:

     * Saving file as .xls will make table viewable in MS Excel (97+), but formatting is not present.
     * No protection is in place to keep user from modifying columns settings in ways that interfere with the program's tabling process.
     * TRminator silently deletes any file at the output path(s); does not warn user that file exists there.
     * TRminator only supports BBF-type documents with "container" objects of type "Model." (Mode 2 of the GUI is an exception).
     * TRminator does not currently support these BBF markup types:
         - Section
         - Hidden
         - Units
         - Key
