Introduction

The following instructions are for the TRminator Graphical Interface.
If you are interested in the command-line interface of TRminator, go here.

Running TRminator with Graphical Interface

There are two ways to run TRminator with the graphical interface.
In most environments, this can be done by double-clicking on the executable jar.
In all environments, the program can be run in Graphical mode via the command line: 
user@host> java -jar TRminator.jar -gui
There are currently no supported command-line arguments.

Common Features to All Modes

Each time a file location (input OR output) or option is changed, the "GO" button is disabled.
Users must click the "Load File"/"Load Files" button in order to enable the GO button.
Each time the Load button is clicked, the program makes sure it can write to the output path, and deletes any file there.
Then the input file(s) go(es) through an in-depth validation process, to ensure that no errors occurr due to a missing file.

NOTICE: because of this validation and possible requirement for completion of the table,
all files associated with the input file(s) via "import" statements MUST be located in top level of the directory where the input file is located.
Attempts to subvert this by using [File > Change Working Directory] will likely result in error.

Menu options

     File > Change Working Directory allows the user to pick a directory where they will choose input files from.
     The actual benefit of this option is quite minimal, as the setting is not guaranteed to be persistent.
     Edit > Change Cols allows the user to edit the columns that exist in the output table.


NOTICE: Changing the columns from the default configuration may have adverse effects, as selected features may depend on the existence of some columns.

Using the Different Modes:

Versioned Document => Table (default)

This mode is for creating an HTML table representing a versioned BBF document's content.
Each version imported (recursively) by the document is included in the table, and bibliographic information is attached in a separate table.
Consequentially,
By default, HTML formatting is used to show the changes since the last version of the document (if applicable).
Users may toggle this feature using the "Diff final two tables" check box under "Misc. Options."
Also available is an option to include a table of profiles.

NOTICE: in the event of a file with more than one model, the user will be prompted to pick the desired model from a list.

General XML => Table

This mode is for creating an HTML table representing an XML document like BBF's TR-069, TR-198 documents.
Unlike the default mode, no inheritance or dependence is validated or included.
Also, the user must specify the name of the object that contains the desired data.

Non-Sequential Documents => Diff Table

This mode is for creating an HTML table representing the differences between a BBF document and an older version of that document.
Essentially, this mode carries out the functions of the default (first) mode twice; once for each of the files, and then compares them.
The resulting table has HTML formatting that shows the difference between the two input files.
