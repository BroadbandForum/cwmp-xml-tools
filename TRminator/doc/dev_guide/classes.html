Versioned (Interface)

Package: threepio.container
Description: The threepio.container.Versioned interface is a simple interface for versioning.
Requirements: Versioned objects will always contain a gettable and setable String, used to name or version it.
Uses: Associating two different objects as being based on the same information, such as a document and table.

Doublet (Class)

Package: threepio.container
Description: The threepio.container.Doublet class is a minimal implementation of java.util.Map.Entry with a modification allowing for the key to be set.
Doublets are often passed to and from methods in Threepio + TRminator in an ML way.
Requirements: none
Uses: Passing and returning two objects as a set, knowing or implying which is the key and which is the value.
Doublets can be helpful when using static methods. The programmer can modify or assign two values instead of one (using the two values returned from the method).

IndexedHashMap (Class)

Package: threepio.tabler.container
Description: The threepio.tabler.container.IndexedHashMap class is a hybrid of HashMap and ArrayList.
Though it techincally extends the ArrayList class, the programmer should treat it more like the HashMap class. It essentially adds the ability to put the value in an indexed location, and get it from it, but also to get values based on a key.
Requirements: use map methods (put, putall), no use of list methods for adding and removing "add" methods (add, addall)
Uses: The Table class extends IndexedHashMap. This allows for indexed rows that also have keys.
